# Visual System — Иркпортал

Краткие принципы редакционной визуальной системы сайта.

## 10 принципов

1. **Один блок — одна сильная городская деталь.** Не перегружать секцию несколькими ракурсами; каждый блок отвечает за один образ или факт.

2. **Фото из архива, не сток.** Использовать опубликованные снимки фотоархива, обложки сущностей CMS и курируемые SVG-fallback из `/public/images/`.

3. **Подпись важнее декора.** У каждого сильного кадра — alt, при возможности год, место, автор или источник.

4. **Спокойные плейсхолдеры.** Без грубых «Фото скоро» — только редакционный тон и текстурный фон.

5. **Единые токены.** Цвета `--color-city-*`, `--radius-card`, `--shadow-soft`, `--image-overlay-soft` в `globals.css`.

6. **Компонент `CityImage`.** Все новые и обновлённые изображения — через `components/visual/city-image.tsx` с `sizes`, `priority` только для hero.

7. **Без layout shift.** Фиксированные aspect-ratio, `next/image`, lazy по умолчанию.

8. **Редакция, не галерея.** Сетки фото — сдержанные, с подписями; без masonry-хаоса и тяжёлых анимаций.

9. **Пустые состояния — по-человечески.** Тексты из `lib/visual-assets.ts` → `VISUAL_EMPTY_COPY`.

10. **Эволюция, не редизайн.** Улучшаем существующие карточки и секции; маршруты, CMS, лиды и админку не ломаем.

## Ключевые файлы

| Файл | Назначение |
|------|------------|
| `app/globals.css` | Design tokens и утилиты |
| `lib/visual-assets.ts` | Fallback-изображения, resolve, empty copy |
| `components/visual/city-image.tsx` | Базовый редакционный кадр |
| `components/visual/city-hero-visual.tsx` | Hero с одним фото |
| `components/visual/route-visual-card.tsx` | Карточка маршрута |

## Разрешение изображения

```
related photo → entity cover → curated fallback → CityImagePlaceholder
```

Только опубликованные фото (`PHOTO_PUBLISHED_WHERE`).
