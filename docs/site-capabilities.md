# Иркпортал — возможности платформы

**Сайт:** https://irkportal.ru  
**Для:** заказчик / владелец проекта  
**Версия:** production, июнь 2026

---

## Зачем этот документ

Это не техническая спецификация, а **обзор того, что уже работает** на сайте: как он помогает собирать заявки, удерживать внимание гостя, экономить ваше время и масштабировать контент без постоянной помощи разработчика.

**Коротко:** Иркпортал — это не «визитка», а **авторская платформа**: маршруты на карте, редакционные материалы, фотоархив, B2B-воронка, сувениры, AR-открытки и единая система заявок — всё связано в одну экосистему.

---

## 1. Главная идея продукта

| Было (типичный сайт гида) | Стало (Иркпортал) |
|---------------------------|-------------------|
| Одна страница «обо мне» | Много разделов с разными сценариями входа |
| Форма «напишите нам» без контекста | Заявка знает, **откуда** пришёл гость и **что** его интересовало |
| Контент правится в коде | Контент в **CMS** — вы сами публикуете |
| Разрозненные Telegram и формы | **Один вход** в админку для всех лидов |
| SEO «как получится» | Sitemap, robots, метатеги, аналитика |

**Итог для бизнеса:** гость приходит с разных точек (карта, статья, сувенир, бизнес-лендинг), а вы видите **полный контекст** и можете отвечать точечно — это повышает конверсию и снижает потери лидов.

## 2. Разделы сайта и что они дают

### Главная `/`

- **Кинематографичный hero** — первое впечатление, настраивается из CMS (тексты, фото автора, CTA).
- **Сценарный выбор** — гость сам указывает интерес: прогулка, бизнес, материалы, сувениры.
- **Блок автора** — доверие и личный бренд Алёны.
- **Превью фото, сувениров, бизнеса** — ведут вглубь сайта, не отпуская на первом экране.
- **Сквозной блок заявки** внизу страницы.

### Маршруты и карта `/map`

- Интерактивная **карта Яндекса** с маршрутами.
- Фильтры: самостоятельные / с гидом / по категориям.
- **Страница каждого маршрута** — описание, точки, обложка, CTA «обсудить».
- Заявка с маршрута **автоматически подставляет** название и slug маршрута.
- Старый URL `/excursions` ведёт на карту с фильтром guided — SEO не теряется.

### Исследовать `/explore`

- Редакционные **материалы о городе** (статьи, история, подборки).
- Категории, featured-материалы, связь со статьёй → маршрут.
- CTA на каждой статье — гость может спросить по конкретному материалу.

### Фотоархив `/explore/photos`

- Публичная витрина городских снимков (старое / новое Иркутск).
- Фильтры: год, улица, автор, категория.
- **Модерация и права** — публикация только после одобрения (защита от юридических рисков).
- **Форма для гостей** `/explore/photos/submit` — краудсорсинг архива с модерацией в админке.

### Для бизнеса `/b2b` → `/business`

- Полноценный **B2B-лендинг**: корпоративы, программы, форматы.
- Расширенная форма: компания, тип задачи, бюджет, формат, даты.
- **Deep links** — с экскурсии или маршрута кнопка «Обсудить дату» открывает `/business` уже с заполненным контекстом в сообщении.
- Старые URL `/program` и `/for-companies` редиректят сюда.

### Сувениры `/souvenirs`

- Каталог **без онлайн-оплаты** — осознанный выбор: заявка, а не «брошенная корзина».
- Карточки товаров, мастера, статусы наличия (в наличии / под заказ / предзаказ).
- Заявка на товар фиксирует **slug, название, категорию**.
- Форма **«Стать мастером»** для расширения каталога.
- `/shop` редиректит на `/souvenirs`.

### AR-открытки `/ar-postcards`

- Витрина интерактивных открыток с QR.
- Страница открытки: видео, аудио, внешний AR-эффект.
- **API генерации QR** для печати и офлайн-продвижения.
- Предзаказ и вопросы через ту же систему заявок.

### О проекте, контакты, клуб

- `/about` — манифест, автор, гиды.
- `/contact` — форма + мессенджеры из настроек сайта.
- Ссылка на **Boosty (Клуб)** в меню «Ещё».

### События `/events`

- Календарь событий (включается, когда есть опубликованные записи).
- Пункт меню **скрывается автоматически**, если событий нет — нет «пустых» разделов.

### Политика конфиденциальности `/privacy`

- Юридическая страница для согласий в формах.

## 3. Система заявок — сердце конверсии

Все формы сайта сходятся в **одну коллекцию «Заявки»** в админке. Это главный «ништяк» для эффективности.

### Единая форма `LeadForm`

Одна умная форма адаптируется под контекст:

| Вариант | Где используется |
|---------|------------------|
| Компактная | Блоки CTA на страницах |
| Полная | Страница контактов |
| B2B | Раздел «Для бизнеса» |
| Товар / сувенир | Карточки сувениров |
| AR | Предзаказ открытки |
| Мастер | Размещение в каталоге |

### Что сохраняется в каждой заявке

- Имя, email, телефон, Telegram, MAX
- **Предпочтительный способ связи**
- **Тип заявки** (маршрут, бизнес, товар, AR, фото…)
- **Источник** — страница, блок, UTM-метки
- Slug и название маршрута / статьи / товара / открытки
- Текст сообщения, даты, размер группы, бюджет (для B2B)
- **Согласие на обработку данных** (текст и версия)
- Приоритет (высокий / обычный / низкий) — автоматически для B2B и заказов

### Точки сбора заявок на сайте

Заявку можно оставить практически **на каждом шагу воронки**:

1. Главная — блок CTA  
2. Каждый маршрут  
3. Карта (индекс)  
4. Каждая статья и раздел «Исследовать»  
5. Фотоархив и страница фото  
6. B2B-лендинг (расширенная форма)  
7. Каждый сувенир и мастер  
8. AR-открытки  
9. Страница «О проекте»  
10. Контакты  

**Плюс:** плавающая кнопка «Связаться» (Telegram / MAX / Email) на всех страницах, кроме карты и контактов — не мешает карте, но всегда под рукой.

### Защита от спама

- **Honeypot** — ловушка для ботов  
- **Минимальное время заполнения** (3 сек) — отсечение автопостов  
- **Rate limit** — не более 5 заявок в минуту с одного IP  
- **Санитизация текста** — защита от вставки HTML  

### Уведомления

- Все заявки в **админке** с фильтрами и статусами (`new` → `in_progress` → `closed`)
- Email-уведомления через **Resend** (после подключения ключа)
- При закрытии заявки — опциональный **запрос отзыва** на email гостя

### Аналитика действий

Клики по мессенджерам и отправки форм отслеживаются для **Яндекс.Метрики** — видно, какой канал и страница работают лучше.

## 4. Эргономика и UX — почему гостю удобно

### Mobile-first

Сайт проектировался **с телефона**: навигация, карта, формы, плавающая кнопка — всё usable на маленьком экране.

### Навигация без перегруза

- **5 главных пунктов** в шапке: Маршруты, Исследовать, Для бизнеса, О проекте, Контакты  
- Остальное — в меню **«Ещё»**: События, Фото, AR, Клуб, Сувениры  
- CTA в шапке: **«Спланировать визит»** → бизнес-воронка  

### Сценарный вход с главной

Гость не гадает «куда нажать» — **Scenario Picker** предлагает 4 понятных пути. Меньше отказов, выше вовлечённость.

### Плавная прокрутка и анимации

- Lenis smooth scroll  
- GSAP-анимации с учётом **prefers-reduced-motion** (доступность)  
- Без агрессивных эффектов — спокойная «редакционная» эстетика  

### Визуальная система

- Единые цвета, тени, радиусы  
- Компонент **CityImage** — быстрая загрузка, без «прыгающей» вёрстки  
- Hero с **реальными фото** из архива, не сток  
- Пустые состояния с человеческими текстами, не «404 контента»  

### Доступность (a11y)

- Семантическая разметка, aria-атрибуты  
- Focus-visible для клавиатуры  
- Контраст и читаемая типографика

## 5. CMS — вы управляете без программиста

**Админка:** https://irkportal.ru/admin

### Dashboard

- Счётчики: маршруты, статьи, товары, **новые заявки**, фото на модерации  
- Быстрые кнопки: создать маршрут, статью, открыть заявки, настройки  

### Коллекции контента

| Раздел CMS | Что редактируете |
|------------|------------------|
| Маршруты | Карта, страницы маршрутов, геометрия |
| Материалы | Статьи «Исследовать» |
| Фото Иркутска | Архив + модерация |
| События | Календарь |
| Экскурсии | Отдельные страницы программ |
| Товары | Сувениры |
| Мастера | Авторы изделий |
| AR-открытки | Витрина и QR |
| Медиа | Все файлы и изображения |
| Заявки | Все лиды с сайта |
| Настройки сайта | Контакты, hero, SEO, согласия |

### Публикация в один клик

Статус **«Опубликован»** — материал на сайте. **«Черновик»** — только в админке.  
После сохранения сайт **обновляется автоматически** (revalidate).

### Безопасность контента на проде

- На production **не показываются** демо-данные из кода — только ваш CMS-контент  
- Фото не публикуются без модерации и подтверждения прав  
- Телефон-заглушка не отображается гостям

## 6. SEO, поиск и аналитика

| Инструмент | Статус | Польза |
|------------|--------|--------|
| **Sitemap.xml** | Работает | Индексация только реальных страниц |
| **Robots.txt** + Host | Для Яндекса | Корректная привязка домена |
| **Google Search Console** | Метатег на главной | Подтверждение и мониторинг |
| **Яндекс.Метрика** | ID 109995467 | Поведение, цели, воронки |
| **Open Graph** | На страницах | Красивые превью в мессенджерах |
| **Canonical URL** | B2B и ключевые страницы | Без дублей в поиске |
| **Редиректы 301/308** | shop, program, excursions | Старые ссылки не бьются |

## 7. Техническая надёжность (для спокойствия заказчика)

- **Production на VPS** с PM2 — автоперезапуск при сбоях  
- Скрипт деплоя: pull → build → restart  
- `npm run check:prod` — проверка ключевых URL после обновлений  
- PostgreSQL + Payload 3 — промышленный стек  
- Sharp для ресайза изображений в админке  
- Формы с Zod-валидацией — меньше мусорных заявок

## 8. Что это даёт бизнесу — итоговая выгода

### Больше заявок

- Много точек входа + контекст в каждой заявке = выше конверсия  
- B2B-лендинг с deep links с маршрутов и экскурсий  
- Плавающий контакт ловит «тёплых» на любой странице  

### Меньше потерь

- Один inbox в админке вместо разрозненных сообщений  
- Email-уведомления (после Resend)  
- Статусы заявок — ничего не забывается  

### Экономия времени

- Контент правите сами в CMS  
- Модерация фото в одном месте  
- Не нужен программист для каждой статьи или маршрута  

### Масштабирование бренда

- Фотоархив как медиа-актив  
- Сувениры и мастера — дополнительная монетизация  
- AR-открытки — PR и «вау»-фактор  
- Материалы «Исследовать» — SEO-трафик по запросам об Иркутске  

### Доверие и юридическая аккуратность

- Согласие на данные в каждой форме  
- Политика конфиденциальности  
- Модерация и права на фото
